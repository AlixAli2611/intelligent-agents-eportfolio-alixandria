Reflective Case Studies

1. Explainable Decisions in Agents

WHAT: I added ranking scores and recommendation labels to make ResearchMate's decisions visible.

SO WHAT: This mattered because retrieved papers are not automatically useful. Scores, labels, and reasons made the system more transparent.

NOW WHAT: In future systems, I would design explainability from the beginning.

2. AI Safety Constraints and Bounded Autonomy

WHAT: Groq/Llama supported reasoning, but Python controlled the workflow, validation, and fallback behaviour.

SO WHAT: This limited the risk of invalid output, API failure, or unsupported claims.

NOW WHAT: I will treat bounded autonomy as a core design feature in future agent systems.

3. Responsible Deployment

WHAT: ResearchMate includes source-level summaries, evidence notes, limitations, and human review reminders.

SO WHAT: This prevents the tool from appearing more authoritative than it is.

NOW WHAT: I will make limitations visible in future AI tools rather than hiding uncertainty.
