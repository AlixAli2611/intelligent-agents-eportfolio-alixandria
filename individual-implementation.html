Individual Implementation: ResearchMate

Section focus: This page explains the final ResearchMate prototype, its workflow, tools, and evidence outputs.

Portfolio Overview

This e-portfolio presents evidence of my learning, development, and applied practice throughout the Intelligent Agents module. The central project is ResearchMate, a multi-agent academic research assistant designed to support the early stages of academic research. The work developed in two phases: first as a group design proposal, and later as my individual implementation of an LLM-powered planning agent.

The individual implementation phase required me to move from design to practice. This was an important shift because the design proposal was ambitious, but the implementation had to be realistic, tested, and demonstrable. My final ResearchMate prototype accepts a user’s topic, purpose, audience level, and requested number of papers. It then creates a plan, retrieves academic sources from arXiv, Semantic Scholar, and OpenAlex, processes abstracts, ranks papers using a scoring rubric, produces an evidence note, and saves outputs as Markdown, JSON, and SQLite records.

Implemented Workflow

1. Goal Input

The user enters a topic, purpose, audience level, and number of papers.

2. Planning

The Planning Agent generates a structured research plan.

3. Retrieval

The Retrieval Agent queries arXiv, Semantic Scholar, and OpenAlex.

4. Processing

The Processing Agent creates summaries, key terms, and paper type labels.

5. Ranking

The Ranking Agent scores papers and assigns recommendation labels.

6. Storage

The Storage Agent saves Markdown, JSON, and SQLite outputs.

Evidence of Work and Artefacts

The first major artefact is the group design proposal. This proposal demonstrates our understanding of intelligent agent architecture and system design. It described ResearchMate as a hybrid multi-agent system using Llama-3.3-70B via Groq, arXiv, Semantic Scholar, OpenAlex, ChromaDB, SQLite, Pydantic, LangGraph, Chainlit, and asyncio. Although not every component was implemented later, the proposal shows the design ambition and reasoning behind the original architecture.

The second artefact is the individual ResearchMate implementation. This includes the Python codebase, agent files, Pydantic models, tests, README, requirements file, output files, screenshots, and GitHub repository. The implementation demonstrates that the project moved beyond design into a functioning prototype that can accept user input, retrieve sources, process results, rank them, and save outputs.
